# 📘 Planejamento dos Estudos e Organização dos Projetos

Hoje não foi um dia focado em escrever código, mas sim em organizar os próximos passos da minha jornada de aprendizado.

## ✅ O que fiz hoje

Passei um tempo analisando a estrutura do meu GitHub e pensando em como organizar melhor meus projetos para que eles representem minha evolução de forma clara.

Cheguei à conclusão de que, neste momento, o melhor caminho é:

- Continuar focada em finalizar o curso **Databases and SQL for Data Science with Python**.
- Manter o repositório **sql-ciencia-de-dados-python** exclusivamente para os estudos de SQL.
- Continuar estudando **Administração de Banco de Dados** em paralelo, mas sem misturar os conteúdos por enquanto.

---

## 💡 Planejamento

Depois de finalizar o curso de SQL, pretendo iniciar um novo projeto voltado exclusivamente para Administração de Banco de Dados.

A ideia é utilizar o banco de dados **EstudosDados**, que desenvolvi durante meus estudos de SQL, como base para aplicar conceitos mais avançados, como:

- Backup e Restore;
- Logs de Transações;
- Segurança;
- Usuários;
- Roles;
- Permissões;
- Auditoria;
- Monitoramento do banco de dados.

Assim, o mesmo banco de dados continuará evoluindo conforme eu aprender novos conceitos.

---

## 🧠 Aprendizado de hoje

Hoje percebi que nem sempre o mais importante é sair criando novos projetos.

Organizar bem os estudos também faz parte do processo.

Ter um planejamento ajuda a manter o GitHub mais organizado e permite mostrar uma evolução contínua, desde consultas SQL até conceitos mais avançados de Administração de Banco de Dados.

---

## 🚀 Próximos passos

Meu foco agora continua sendo:

- Finalizar o curso **Databases and SQL for Data Science with Python**;
