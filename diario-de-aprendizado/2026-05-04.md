# 📘 Administração de Banco de Dados Relacional (DBA) — Laboratório

Hoje continuei praticando o laboratório do curso e acabei aprendendo uma coisa muito importante na prática 😅

## ✅ O que fiz

Tentei recriar o banco de dados:

sql

CREATE DATABASE world;

O sistema informou que o banco já existia.

Então entendi que não precisava criar novamente, apenas acessar o banco.

Usei:

USE world;

Depois executei o script para finalizar a configuração do banco:

SOURCE world_mysql_script.sql;

Esse script basicamente cria as tabelas e carrega os dados automaticamente dentro do banco.

⸻

🧠 Meu entendimento

Percebi que:

* Nem sempre precisamos criar o banco novamente
* Se ele já existir, basta selecionar com USE
* O comando SOURCE funciona como uma instalação automática do banco de dados

Foi interessante porque comecei a entender melhor como um DBA realmente trabalha no dia a dia, executando scripts e organizando o ambiente antes de começar a usar os dados.

⸻

🤔 Dúvida que surgiu

Algo que ainda estou me acostumando:

👉 sempre que volto para o terminal preciso selecionar novamente o banco com USE nome_do_banco.

Agora entendo que o servidor pode ter vários bancos e eu preciso dizer em qual quero trabalhar.

⸻
💡 Aprendizado importante

Hoje também aprendi algo na prática:

Se o laboratório cair ou a internet desconectar, o ideal é anotar ou salvar os comandos, porque o ambiente pode reiniciar.

Isso já me fez pensar mais como alguém que administra bancos de dados de verdade
