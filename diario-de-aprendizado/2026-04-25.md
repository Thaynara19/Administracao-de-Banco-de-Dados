# 📘 Administração de Banco de Dados Relacional (DBA)
## Módulo 1 — Segurança, Conformidade e Fundamentos do DBA

Hoje avancei nos estudos do curso **Administração de Banco de Dados Relacional (DBA)** pelo Coursera.

Nesta etapa comecei a entender melhor o papel do DBA além da parte técnica, principalmente relacionado à segurança de dados, conformidade e responsabilidade profissional.

## 🔐 Segurança de Dados

Aprendi que trabalhar com dados exige mais do que conhecimento técnico.  
É necessário garantir:

- Transparência no uso das informações
- Consentimento do usuário
- Integridade e proteção dos dados

Percebi que o DBA atua em conjunto com analistas, desenvolvedores e áreas de negócio, sendo importante também manter registros e documentações para garantir segurança e responsabilidade no acesso aos dados.

## 🛡️ Projeto de Sistemas Seguros e Conformidade

Estudei a importância de projetar sistemas já pensando em segurança desde o início.

Alguns pontos importantes:

- Controle de permissões de acesso
- Princípio do menor privilégio
- Boas práticas de conformidade
- Proteção contínua das informações

## 🧠 Rotina do DBA

Durante as atividades avaliativas, revisei conceitos importantes:

- Monitoramento do banco através de indicadores de desempenho
- Uso de tíquetes de suporte para comunicação com usuários
- Ciclo de vida dos dados
- Análise de requisitos e modelagem de banco de dados

Essa etapa ajudou a entender melhor como funciona o dia a dia real de um Administrador de Banco de Dados.

## ✅ Atividade avaliativa - 25/04/2026

Durante as avaliações do módulo, consegui reforçar alguns pontos importantes:

- O DBA trabalha junto com desenvolvedores, engenheiros de dados e arquitetos para definir decisões do banco, como a plataforma utilizada.
- A modelagem de banco de dados é essencial na fase de Design e Planejamento.
- A etapa de Monitoramento e Manutenção também envolve automação de tarefas repetitivas.
- Transparência significa deixar claro para o cliente quais dados serão coletados e como serão utilizados.
- Conformidade envolve compreender e seguir regulamentações que impactam a organização.

🧠 Essas atividades ajudaram a consolidar melhor minha visão sobre segurança, governança e responsabilidade no trabalho de um DBA.
