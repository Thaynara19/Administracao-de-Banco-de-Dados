# 📘 Dia 01 — Introdução à Administração de Banco de Dados (DBA)

## ✅ O que aprendi

Neste início do curso de Administração de Banco de Dados Relacional (DBA), compreendi que o trabalho do DBA vai além do uso de SQL.

O administrador de banco de dados é responsável por acompanhar todo o ambiente de dados, garantindo que os sistemas funcionem corretamente.

Entre as principais atividades estudadas:

* Monitoramento diário dos bancos de dados
* Verificação de backups e tarefas automatizadas
* Atendimento de solicitações e tickets de usuários
* Participação no planejamento de novos bancos de dados
* Análise de relatórios e desempenho do sistema

Também aprendi que o banco de dados possui um ciclo de vida composto por quatro etapas:

1. Entender a necessidade do sistema
2. Planejar e projetar o banco de dados
3. Implementar a estrutura
4. Monitorar e manter o funcionamento continuamente

## 🧠 Entendimento pessoal

Passei a enxergar o DBA como o profissional responsável por acompanhar continuamente o funcionamento do banco de dados, analisando relatórios para garantir estabilidade, segurança e bom desempenho ao longo do tempo.
